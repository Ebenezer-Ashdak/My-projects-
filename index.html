Bluetooth + Wi‑Fi (WebRTC) P2P Chat — Demo

This page demonstrates two transport options:

  • WebRTC (Wi‑Fi / hotspot) — Peer‑to‑peer chat using WebRTC. Works well when both devices are on the same Wi‑Fi or one device is a hotspot. Signaling is manual (copy/paste) in this demo.
  • Bluetooth (BLE) — Connect to a BLE peripheral device and read/write a text characteristic. Important: browsers cannot act as BLE peripherals, only as centrals. That means two phones using only their browsers cannot reliably chat over Bluetooth without a native/peripheral role on one side (e.g., microcontroller or an app).

WebRTC (Wi‑Fi / Hotspot) — Peer chat

Use this if both devices are on the same network (including one device creating a hotspot). This demo uses manual signaling: create an offer, copy the SDP to the remote device and paste the answer back.

Tip: Use one device to create an offer and share the generated SDP with the other device (by copy/paste, QR, etc.). Then paste the answer back into the offerer.

Bluetooth (BLE) — Connect to a peripheral

This connects your browser as a central to a Bluetooth LE peripheral. The peripheral must expose a writable characteristic that accepts text. Use apps like nRF Connect on another phone, or a microcontroller (ESP32, Nordic) set up as a BLE peripheral to test.



Note: Browser BLE support varies. Chrome on Android supports Web Bluetooth but cannot advertise or act as peripheral.

Testing notes: For Wi‑Fi/hotspot chat, open this page on both devices. Use 'Create Offer' on one, copy the Local SDP to the other, click 'Create Answer' there, copy that answer back, and click 'Apply Remote SDP' on the first. For BLE, run a BLE peripheral that offers the given service/characteristics (the values default to Nordic UART Service UUIDs — many BLE UART peripherals use these).
